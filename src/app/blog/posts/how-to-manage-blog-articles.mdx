## Introduction

Managing your blog articles is simple and straightforward. This guide covers everything you need to know about creating new articles and removing old ones from your portfolio application.

## Part 1: Creating New Blog Articles

### Step 1: Locate the Blog Posts Directory

Navigate to your project structure:

```
your-project/
└── src/
    └── app/
        └── blog/
            └── posts/  ← Go here
```

### Step 2: Create a New MDX File

Create a new file with the naming format: `your-article-title.mdx`

**Naming conventions:**
- Use lowercase letters
- Use hyphens to separate words
- Be descriptive but concise

**Examples:**
- ✅ `how-to-learn-react.mdx`
- ✅ `typescript-advanced-techniques.mdx`
- ✅ `database-optimization-guide.mdx`
- ❌ `MyArticle.mdx` (avoid uppercase)
- ❌ `article_name.mdx` (avoid underscores)

### Step 3: Add Article Metadata

At the very top of your MDX file, add the frontmatter (metadata) enclosed in three dashes:

```mdx
---
title: "Your Article Title Here"
publishedAt: "2026-05-06"
summary: "Brief description of your article - this shows in blog listings"
image: "https://images.unsplash.com/photo-xxx"
tag: "Category or Tag"
category: "Learning"
---
```

### Step 4: Understand Metadata Fields

| Field | Description | Example | Required |
|-------|-------------|---------|----------|
| **title** | Your article's full title | "React Hooks in 2025" | ✅ Yes |
| **publishedAt** | Publication date (YYYY-MM-DD format) | "2026-05-06" | ✅ Yes |
| **summary** | Short description (50-100 words) for listings | "Master advanced React patterns..." | ✅ Yes |
| **image** | Cover image URL | "https://images.unsplash.com/..." | ❌ Optional |
| **tag** | Main topic/category | "Learning" or "Technical" | ✅ Yes |
| **category** | Article category type | "Learning & Tutorials" | ✅ Yes |

**Tag/Category Examples:**
- Technical
- Learning
- Learning & Tutorials
- Market Trends
- Best Practices
- Advanced Patterns

### Step 5: Write Your Article Content

Below the metadata (after the closing `---`), write your content using Markdown:


## Part 2: Removing Blog Articles

### Step 1: Locate the Article to Remove

Go to: `src/app/blog/posts/`

List of your current articles:

```
src/app/blog/posts/
├── how-databases-actually-work.mdx
├── how-git-actually-works.mdx
├── how-video-calling-works.mdx
├── react-hooks-guide.mdx
└── typescript-guide.mdx
```

### Step 2: Delete the MDX File

**Simply delete the `.mdx` file** of the article you want to remove.

**Example - Removing these articles:**

```
src/app/blog/posts/
├── how-databases-actually-work.mdx  ← DELETE THIS
├── how-git-actually-works.mdx       ← DELETE THIS
├── how-video-calling-works.mdx
├── react-hooks-guide.mdx
└── typescript-guide.mdx
```

**How to delete:**

**On Windows (File Explorer):**
1. Navigate to `src/app/blog/posts/`
2. Right-click the file
3. Select "Delete"

**On Mac (Finder):**
1. Navigate to `src/app/blog/posts/`
2. Right-click the file
3. Select "Move to Trash"

**Using VS Code:**
1. Open the file in VS Code
2. Right-click the filename in sidebar
3. Select "Delete"

**Using Command Line (Git Bash/Terminal):**
```bash
rm src/app/blog/posts/your-article-name.mdx
```

### Step 3: Verify Removal

Once deleted:

✅ Article disappears from `/blog` listing  
✅ Individual page becomes 404 Not Found  
✅ Won't appear in related articles anymore  
✅ No broken links in your application  

### Step 4: Git Commit (Optional)

If using Git version control:

```bash
# See deleted files
git status

# Stage the deletion
git add -A

# Commit the change
git commit -m "Remove outdated blog articles"

# Push to remote (if using GitHub/GitLab)
git push origin main
```

## Best Practices

### When Creating Articles

1. **Write compelling summaries** - Users see these in listings
2. **Use clear headings** - They auto-generate table of contents
3. **Add relevant images** - Makes articles more engaging
4. **Pick accurate tags** - Helps with SEO and organization
5. **Proofread before saving** - Small changes require rebuilding
6. **Use consistent formatting** - Makes content professional

### When Removing Articles

1. **Keep important articles** - Remove only truly outdated content
2. **Check for backlinks** - Make sure other pages don't link to it
3. **Update related articles** - If something references the deleted article
4. **Commit with descriptive messages** - Document why you removed it

## Troubleshooting

### Article Not Showing Up?

1. Check file location: `src/app/blog/posts/your-file.mdx`
2. Verify `.mdx` extension (not `.md`)
3. Ensure metadata is formatted correctly (three dashes above and below)
4. Rebuild/refresh your application

### Article Shows Errors?

1. Check markdown syntax
2. Ensure code blocks have closing backticks (`)
3. Verify metadata doesn't have special characters without quotes
4. Check image URLs are valid

### Still Have Issues?

- Check the browser console for errors
- Look at build logs
- Verify file names match the URL structure
