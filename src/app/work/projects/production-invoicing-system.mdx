## Overview

A production-grade invoicing system built for real-world financial workflows. The frontend handles complex form logic with dynamic engagement tracking, while the backend manages relational invoice data with NestJS and Prisma.

## Key Features

- **PATCH API Integration**: Dynamic payload construction for partial invoice updates with optimistic UI patterns
- **Max-Amount Fill Distribution**: Intelligent distribution algorithm across invoice line items
- **Engagement Total Tracking**: Real-time total calculations with double-entry bug prevention
- **Relational Data Modeling**: NestJS + Prisma backend with properly normalized invoice schema

## Technologies Used

- **React**: Complex form state management with controlled components
- **TypeScript**: End-to-end type safety for financial calculations
- **NestJS**: Modular backend architecture with dependency injection
- **Prisma**: Type-safe ORM for PostgreSQL with migration support
- **PostgreSQL**: Reliable relational database for financial data

## Challenges and Learnings

Managing complex form state with interdependent fields required careful architecture. Double-entry tracking bugs were resolved by implementing a single source of truth for engagement totals with derived calculations.

## Outcome

The system is actively used in production, processing invoices with zero calculation errors and significantly reducing manual data entry time.
